# Visual QA Checklist — Polar_Bigdata

> 모든 그림/지도/3D/애니메이션은 export 후 이 체크리스트로 검토하고, 결함 수정 후에만 "완료".
> 리뷰는 `scientific-figure-reviewer`(과학) + `visual-reviewer`(디자인) 에이전트로.

## A. 메시지·구조
- [ ] 하나의 지배 메시지가 3초 안에 읽히는가? 제목이 결론형인가?
- [ ] 비전문가가 "이게 공간적으로 어떻게 생겼나"를 이해하는가?

## B. 과학적 정확성
- [ ] 축·컬러바·주석에 **단위**가 있는가? (cm, °C, cm/yr, km)
- [ ] 부호장은 0중심 발산(vik/broc), 비음수는 순차형(oslo_r/acton/davos_r)인가?
- [ ] 컬러 범위(vmin/vmax)가 애니메이션/비교 그림 간 **고정**인가?
- [ ] RMSE 옆에 R²·target_SD·skill-over-mean 병기(성능 그림)?
- [ ] AOA 외삽영역이 회색 마스킹되어 있는가(전이/외삽 지도)?
- [ ] ALT(계절 최대융해) vs MAGT(연평균장) 혼동 없는가?
- [ ] 데이터 근거(n, CV종류, scope)가 캡션에 있는가?

## C. 디자인·가독성
- [ ] 냉색 규약 준수(붉은 계열/rainbow/jet 없음)?
- [ ] 타이포 스케일 일관(제목15/축11/눈금9), 인쇄 크기에서 가독?
- [ ] 격자 정렬, 여백 ≥8%, 상/우 spine 제거?
- [ ] 색으로만 정보 전달하지 않는가(색맹안전, 마커/해칭 병행)?
- [ ] 지도 종횡비=물리비율, 스케일바/격자?

## D. 내보내기
- [ ] 벡터(SVG/PDF) + 고해상 PNG 둘 다?
- [ ] 논문/포스터 그림에 래스터 텍스트 없는가?
- [ ] 파일이 outputs 규칙 폴더에 서술형 이름으로 저장?

## E. 시리즈 일관성 (관련 그림 묶음)
- [ ] 같은 양은 같은 컬러맵·범위·단위로 표현?
- [ ] 선폭/마커/폰트가 묶음 전체에서 통일?
