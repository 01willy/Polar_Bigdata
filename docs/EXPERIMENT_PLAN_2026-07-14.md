## B. 개인 궁금증 4문항 답 (Q1~Q4)

### Q4. Gautam 2025가 정답 68곳뿐인데 연구 의미가 있었나
있다. 두 가지 의미다. (1) **적은 데이터로도 "무엇이 ALT를 지배하나"(연평균기온 19%·경사 18%)를 밝히고**, 알래스카 전역 지도로 외삽했다. (2) 더 중요한 건 **"순수 ML이 라벨 68개에서 과적합해 무너진다(train R² 0.84→test 0.24), 물리모델(Stefan)이 더 견고(0.54)"를 정직하게 보고**한 것. 즉 "적은 데이터의 한계"를 규명한 것 자체가 기여다. 우리 14,348곳은 그보다 200배 많아 더 유리하다. 교훈: 라벨 희소 시 물리 결합(P2)이 정석 처방.

### Q1·Q2. 공간 이미지 + 조건(Stefan·시추공·적설) → ALT+3D 등온장 생성모델 (diffusion/flow), GPU 감당 가능한가, 데이터 충분한가
- **시도 가치 있다.** GPU 4장이면 연산은 충분히 감당한다. 병목은 연산이 아니라 **image-label 쌍의 수**다.
- 데이터 현실: ALT 라벨 14,348곳(알래스카), 시추공 260사이트. **image-conditioned diffusion/flow는 통상 수만~수십만 쌍**을 원하는데, 우리는 그 하단이라 **from-scratch 학습은 과적합 위험**이 크다. 앞서 신경장이 GBM에 진 것도 이 때문.
- **현실적 설계(권장)**: (a) 순수 생성모델을 처음부터 학습하지 말고, **파운데이션 임베딩(AlphaEarth 64차원)을 조건으로 넣은 경량 diffusion/flow**로 라벨 희소성을 우회. (b) **패치 크기·해상도를 조절**(예: 32×32 저해상 다수)해 쌍 수를 늘림. (c) Stefan 물리 출력을 조건(conditioning)으로 주면 데이터 부족을 물리가 보완. → 이게 P2(Stefan+DL)·P4(임베딩)와 자연스럽게 합쳐진다.
- **판정**: 이미지 CNN/diffusion은 **P4 이후 연구 트랙으로 시도**하되, 사전등록 게이트(공간블록 CV로 GBM 이겨야 채택)로 과적합을 통제. 안 되면 "라벨 희소로 무겁게 못 간다"는 정직한 음성 결과.

## C. 실험 계획 (GPU 6,7,8,9) — 다음 세션 착수 순서

| # | 실험 | 목적(해결 질문) | 산출·게이트 |
|---|---|---|---|
| **P0** | 데이터 인벤토리 세계지도(지역×데이터 매트릭스) + 6모델별 ALT 예측·오차 지도 | Q5·Q9 시각화. 즉시(CPU) | 발표·데이터관리용 그림 |
| **P1** | 전 공변량(DEM+InSAR+PolSAR+CCI 전부) + 전 지역(알래스카+시베리아 ALLena+티베트 QTEC) 통합 ALT 재학습, 6모델 재비교 | Q4·Q6·Q10 "데이터 최대활용·제대로 학습" 정면. 통합 vs 알래스카특화 정직 비교 | 공간블록·LORO 게이트. 개선시 채택 |
| **P2** | Stefan 물리 base + DL 잔차(2단계: 물리 사전학습→시추공 파인튜닝) | Q8 비전 심장, 전이 방어 | 순수 DL/물리 대비 개선 게이트 |
| **P3** | 3D 지중온도에 전 공변량 투입 + 연속성 DL → 매끄러운 0°C 등온면 tri-mesh | Q2·Q4 등온면 끊김 해결 | 심도별 RMSE + 물리 일관성 |
| **P4** | AlphaEarth 파운데이션 임베딩을 공변량 추가 | 정보량 확충(근거 24%↓) | 공간블록 CV로 GBM 이겨야 채택 |
| **P5(트랙)** | 이미지 조건 diffusion/flow(패치+임베딩+Stefan 조건) → ALT+3D 생성 | Q1·Q2 시도. 고위험 | 사전등록 게이트, 과적합 통제 |

**착수 순서**: P0(즉시) → P1(핵심) → P2 → P3/P4 → P5(트랙).
**시각화 원칙(다음 세션)**: 모든 실험 결과는 전문 mapping(지도+오차+불확실성)으로 도시하고 PPT 반영. 냉색 규약(oslo_r/vik/acton), Pretendard, EMP 톤 그래프.

## F. ★자료 구조 발견 (2026-07-14 추가) — ALT와 지중온도는 별개 네트워크

- **ALT 라벨(2D)** = CALM/ABoVE 프로브 관측, **94% 알래스카**(14,348셀).
- **지중온도 라벨(3D)** = GTN-P 시추공, **9개국 260사이트**(러시아115·서시베리아40·미국(알래스카)40·스위스33 등, 스위스+러시아 72%).
- **둘 다 있는 곳(10km내)**: 지중온도 260사이트 중 **36개만** ALT 있음. **단 알래스카는 40중 35가 겹침**(알래스카는 둘 다 있음). 스위스는 지중온도만·ALT 없음(기후는 전지구라 있음).
- **현 상태**: ALT 모델(알래스카)과 3D 지중온도 모델(스위스·러시아 중심)이 **학습 지역이 어긋남.** 별개 처리 중.
- **통합 열쇠**: ALT = 지중온도 0°C 통과 깊이. **시추공 지중온도에서 0°C 깊이를 유도해 러시아·시베리아·유럽까지 ALT 라벨 확장** → 두 목표를 같은 지역에서 일관 학습.
- **P1·P3 반영**: (1) 지중온도→ALT 라벨 유도로 다지역 ALT 확장, (2) 알래스카(둘 다 있음)를 ALT↔지중온도 일관성 검증 기준지로, (3) 3D 스위스 편중이 알래스카 툰드라에 전이되는지 정직 검증.
- **공변량 커버리지 주의**: 기후(ERA5)·지형(DEM)은 전지구 = 어디든 예측 가능. 하지만 **InSAR/PolSAR는 알래스카만** → 타 지역 예측 시 이 두 축은 결측. 다지역 학습 시 모달리티 결측 처리 필요.
