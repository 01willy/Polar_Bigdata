# 연구 프로그램 (2026-07-17) · 엔진 고도화·데이터 완전활용·완벽검증을 백본으로

시간 여유 있음. 예선(07-31)은 중간 체크포인트로 두고, 기존 로드맵(물리 base 고도화·E1 다지역 전이·3D 고도화)을
지금부터 병행한다. 위에 시계열(B1)·크리깅(B2)을 얹는다.

## 0. 연구 구조 요지 (역할 분리)

이 프로그램은 물리를 두 역할로 분리한다. 백본은 라벨 증강이고, 물리-ML 결합은 추가 실험이다.

- **역할 1 (백본): 물리 = 라벨 공급원.** 물리는 ML 훈련용 라벨을 늘리는 수단이다.
  라벨 증강 경로는 세 가지다.
  1. 실측 ALT (시추공·현장 관측): 1급 라벨.
  2. 3D 지중온도장의 0°C 통과 깊이: 온도 프로파일에서 유도한 ALT 라벨(3D→ALT 라벨 증강).
  3. 물리식 예측(Stefan·Kudryavtsev): 라벨 희소 지역·심부에 대한 물리 유도 라벨.
  이렇게 확장한 라벨 집합으로 ML을 훈련·강화한다. 전이(다지역) 시에는 지역 비율·데이터량을
  조절해 표본편향을 통제한다(예: 특정 지역만 증강, 증강량 상한). **이 라벨 증강 경로가 프로그램의 백본(W1)이다.**
- **역할 2 (추가 실험): 물리-ML 결합.** 잔차 학습·파라미터 예측·물리 제약(단조성·함수형태)·AOA 게이팅은
  백본 위에 얹는 추가 실험이다(W2). 백본(라벨 증강)이 성립한 뒤, 물리를 입력·제약으로 넣어 전이 취약을
  보완하는지 게이트로 검증한다.

주의: 라벨 증강은 라벨 품질·표본편향이 곧바로 성능 왜곡으로 이어진다. 유도 라벨(3D 0°C·물리식)은
held-out 실측으로 검증하고, 증강 지역·비율은 사전등록한 게이트로 통제한다. 무통제 증강은 P2에서
관측된 "라벨 증강이 해가 된다"는 결과의 재발 위험이 있으므로, 증강 여부·범위는 통제군 대비로만 채택한다.

## 1. 확정 전제 (이 프로그램의 제약)
- **정보병목**: 6모델 동률. 한계는 모델이 아니라 공변량 정보·지역 다양성. → 지렛대 = 정보 확충·물리, 모델 교체 아님.
- **물리 우선 전이(P2)**: Stefan 아핀(a+E√TDD) LORO 18cm ≫ 순수 ML 40cm. 잔차학습 무익. → 물리를 1급 시민으로.
- **결측 라우팅 아티팩트(P1)**: InSAR/PolSAR 알래스카 한정. 다지역서 "결측=깊은 ALT" 오학습. → 결측 처리 재설계 필수.
- **평가 정직성**: "16.95cm 전이"는 실제 알래스카 내 공간블록 CV. 진짜 다지역(시베리아) 전이는 어렵다. → 두 지표 항상 구분.
- **큐레이션 12.97 = 범위축소 아티팩트**. 절대 RMSE에 R²·skill 항상 병기.
- **선행연구 비교 한계**: 우리가 직접 재채점한 건 ESA CCI뿐. → 우리 데이터에서 재현 가능한 것만 통제 비교.

## 3. 워크스트림 (가설·방법·게이트·산출)

순서는 역할 분리(0절)를 따른다. W1(라벨 증강 백본)이 최상단이고, W2~는 그 위의 데이터·물리·전이·검증 작업이다.

### W1. 라벨 증강 백본 · 물리 = 라벨 공급원 (프로그램의 척추)
물리를 라벨 공급원으로 써서 ML 훈련 라벨을 확장한다. 세 경로(실측 ALT · 3D 0°C 깊이 · 물리식 예측)를
통합 관리하고, 증강 지역·비율·데이터량을 통제군 대비로 조절한다. 상세 실행 항목은 3D(W4.3)·전이(W5.1) 절과 교차한다.
- **W1.1 라벨 소스 통합·품질 관리(QC)**. 가설: 검증된 유도 라벨이 라벨 희소 지역의 ML skill을 올린다.
  - 소스: (a) 실측 ALT(시추공·현장, 1급), (b) 3D 지중온도장 0°C 통과 깊이(유도), (c) 물리식(Stefan·Kudryavtsev) 예측(유도).
  - 방법: 유도 라벨은 held-out 실측 ALT로 δ 검증, 0-400cm 클립, AOA 게이팅으로 적용범위 밖 배제.
  - **게이트**: 유도 라벨 추가가 공간블록+LORO에서 실측-only 대비 skill 개선(사전등록). 무익·악화면 정직한 음성.
  - 산출: 통합 라벨 테이블(소스·신뢰도·QC 플래그), 라벨소스별 Δskill 표.
- **W1.2 증강 비율·지역 통제**. 가설: 증강 지역·비율을 조절하면 전이 표본편향을 줄인다.
  - 방법: 지역별 증강 상한, 증강 비율 스윕, 다지역 LORO에서 편향-분산 곡선. P2 "무통제 증강 악화" 재발 여부 확인.
  - **게이트**: 다지역 LORO에서 증강 통제본이 무증강·무통제증강 대비 개선.
  - 산출: 증강 비율-전이 RMSE 곡선, 채택 증강 정책.

### W2. 데이터 완전 활용 · 정보병목 정면돌파
- **W2.1 미투입 모달리티 확충**. 가설: 새 정보원이 LORO skill을 올린다.
  - SoilGrids(토양: 점토·유기탄소·용적밀도, ISRIC vsicurl 재시도/사전 타일 캐시) → sg_* 피처.
  - Sentinel-1(레이더 후방산란·간섭)·Sentinel-2(NDVI·분광, GEE 또는 사전 타일) → 식생·습윤 대리.
  - AlphaEarth 파운데이션 임베딩 64d → 다중모달 압축 정보.
  - 방법: 셀에 부착 → overnight_cell ablation의 M6/M7 슬롯 채움 + 신규 M10(임베딩).
  - **게이트**: 공간블록+LORO에서 GBM 대비 개선해야 채택(사전등록). 무익하면 정직한 음성.
  - 산출: `alt_ablation_cell_results.csv` 확장, 모달리티별 Δskill 표.
- **W2.2 결측 모달리티 재설계**(P1 아티팩트 교정). 가설: 결측을 학습 신호로 다루면 전이 파탄이 준다.
  - 방법: (a) 모달리티 드롭아웃(학습 중 무작위 결측), (b) 마스크 임베딩(결측 지시자를 명시 입력), (c) 모달리티별 전문가+게이팅.
  - **게이트**: 다지역 LORO(레나)에서 중앙값 대체 대비 개선.
  - 산출: 결측처리 비교표(레나 전이 RMSE), `unified_tournament` 재실행.
- **W2.3 SAR 심층 활용**. 현재 InSAR/PolSAR를 스칼라 5·3개로 증류. 패치·다중시기 침하 표현이 정보를 더 주는지 검토(게이트 통과 시만).

### W3. 물리결합 엔진 (추가 실험) · 백본 위의 결합·제약
역할 2에 해당하는 추가 실험이다. W1 라벨 증강 백본 위에서 물리를 입력·제약으로 결합한다.
- **W3.1 물리 base 고도화**. Stefan(a+E√TDD) → **Kudryavtsev**(토양수분·유기물층·적설 열저항 포함). 가설: 더 풍부한 물리가 전이 base를 낮춘다.
  - 방법: fold별 물리 파라미터 적합, 공간블록+LORO 채점. P2 PHYS_only(18cm)를 baseline으로 개선 여부.
  - **게이트**: LORO에서 Stefan 아핀 18cm 대비 개선.
- **W3.2 물리-ML 결합 재설계**(잔차 대신). 가설: 물리를 제약·입력으로 넣으면 ML의 전이 취약을 막는다.
  - 후보: (a) 물리 예측을 입력 피처(P2 잔차와 다름), (b) 단조성·물리 함수형태 제약(monotonic GBM/√TDD 강제), (c) AOA 게이팅 하이브리드(적용범위 안=ML, 밖=물리로 앵커).
  - **게이트**: 공간블록·LORO 둘 다에서 PHYS_only·ML_only 동시 개선(P2 RESIDUAL 실패 극복 기준).
- **W3.3 불확실성·적용범위 통합**. conformal(CQR)+AOA를 물리결합 모델에 이식. 전이 구간 커버리지 보정, 외삽 경고.
  - 산출: 예측값·90%구간·AOA 마스크 3종 세트(물리결합판).

### W4. 3D 지중온도장 · 완벽 검증 + 고도화
0°C 통과 깊이는 W1 백본의 라벨 소스(b)를 공급한다(W4.3 → W1.1).
- **W4.1 (A1) 공간블록+LORO 재평가**. site-GKF 누설 제거 후 지형+CCI 심부 이득이 진짜인지 확정.
  - 방법: p2_field를 공통 CV(block=floor(lat/0.5)…, LORO region)로 재실행. field-ALT δ held-out, 유도라벨 0-400cm 클립.
  - **게이트**: 누설 통제 후에도 FIELD_full > FIELD_base면 채택.
- **W4.2 3D 고도화**. 깊이 연속성·물리 단조성(심부로 갈수록 진폭 감쇠) 제약, 0°C 등온면 tri-mesh 매끄럽게. PyVista 열큐브.
- **W4.3 3D→ALT 라벨 증강**(A2 후, W1 백본 공급). 시추공 0°C 통과 깊이로 다지역 ALT 라벨 확장. GTNPenv AOA 게이팅 적용. 산출 라벨은 W1.1로 편입.

### W5. 다지역 전이 (E1)
- **W5.1 (A2) GTNPenv 게이팅 재평가**. Lena 3,037셀만 증강, 심부 교란 제거 후 전이 재측정. 증강 정책은 W1.2와 공유.
- **W5.2 라벨 정비·확대**. 시베리아·티베트 라벨 품질(QC)·범위 확대. 스발바르·러시아 GTN-P 정비. 산출 라벨은 W1.1로 편입.
- **W5.3 진짜 다지역 LORO 벤치마크**. W1(라벨 증강)·W2(데이터)·W3(물리엔진) 성숙본으로 시베리아·티베트 held-out 전이 확정. 헤드라인 = 이 값.

### W6. 새 방향 (피드백) · 엔진·데이터 성숙 후/병행
- **W6.1 timelapse ALT**. `alt_era5_temporal.csv`(연도별 forcing) + 물리결합 엔진으로 2005-2024 연별 ALT.
  - 평가: temporal holdout(뺀 연도) + **within-site 시간 skill 분리**(연 anomaly·detrended RMSE)로 "시간 예측분 vs 공간 지배분" 정량화. 근거: T-lite에서 gbm_annual 15.86cm(시퀀스 DL 아님).
  - 산출: 연별 ALT 지도 애니메이션, 추세·변동 귀속.
- **W6.2 co-kriging vs 위성-ML**. 사다리: OK(ALT만)/co-kriging·KED·RK(ALT+위성 2차변수)/GBM/물리, 공간블록+LORO.
  - 프레이밍: 전이(LORO)서 크리깅은 관측 없는 지역 불가 → 위성융합·물리 전이 우위 정량화. pykrige·gstools.

### W7. 정직 평가·산출 (횡단, 상시)
- **W7.1** 평가 프레이밍 정정: 공간블록(지역 내)↔LORO(다지역 전이) 명확 구분. 헤드라인 이중 표기. 상세는 `docs/EVAL_FRAMING_NOTE.md`.
- **W7.2** 통제 비교: 우리 데이터·프로토콜에서 재채점 가능한 것(ESA CCI)만 '통제'로 직접 비교. co-kriging·물리는 우리 데이터 내 자체 비교. Whitcomb 등 선행연구는 보고값+프로토콜 병기(통제 아님). "통제 비교" 표현을 실제 범위로 한정.
- **W7.3** 재현·시각화·문서: 냉색 규약, 지도+오차+UQ 3종, 애니메이션. 코드·model card.
