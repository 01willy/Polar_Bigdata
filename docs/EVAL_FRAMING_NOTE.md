# 평가 프레이밍 정정 노트 (2026-07-17)

이 노트는 모든 보고·발표·문서에서 평가 지표를 일관되게 표기하기 위한 기준이다.
핵심은 두 가지다. (1) '공간블록 CV'와 '다지역 전이(LORO)'는 서로 다른 지표다.
(2) 선행연구 비교에서 '통제'라 부를 수 있는 것은 우리가 직접 재채점한 것뿐이다.

## 1. 두 평가 지표의 구분

| 항목 | 공간블록 CV (지역 내) | 다지역 전이 (LORO) |
|---|---|---|
| 정의 | 한 지역(주로 알래스카) 내에서 위치를 공간블록으로 분할, 미관측 위치 예측 | 지역 하나를 통째로 held-out, 다른 지역으로 학습해 그 지역 예측 |
| held-out 단위 | 지역 내 미관측 위치(공간 블록) | 지역 전체(region-out) |
| 측정 대상 | 같은 기후·지형 분포 내 공간 보간 성능 | 분포 밖(다른 기후·지형) 일반화·전이 성능 |
| 난이도 | 상대적으로 쉬움(분포 내) | 어려움(분포 이동) |
| 대표 수치 | **16.95cm (알래스카 내 공간블록)** | 물리(Stefan 아핀) ~18cm / 순수 ML ~40cm (시베리아 방향 전이) |
| 성격 | in-domain | out-of-domain (진짜 다지역) |

주의: 16.95cm은 알래스카 내 공간블록 CV 값이다. 다지역 전이 값이 아니다.
과거 문서에서 "16.95cm 전이"로 표기된 것은 프레이밍 오류이며, 정확히는 "알래스카 내 공간블록 CV"다.

## 2. 표준 표기 규칙

모든 헤드라인 수치는 두 지표를 라벨로 구분해 병기한다.

- 표기 형식: **"공간블록 16.95cm / 다지역 LORO(시베리아) ~물리 18·ML 40cm"**
- 단독 수치 금지: "우리 오차 16.95cm" 같은 라벨 없는 단독 표기는 쓰지 않는다.
- 지역 명시: LORO 값은 held-out 지역을 괄호로 명시한다(예: LORO(시베리아), LORO(레나)).
- 절대 RMSE 단독 금지: 큐레이션·범위축소 아티팩트(예: 12.97cm)를 피하기 위해 절대 RMSE에 R²·skill을 항상 병기한다.

## 3. 선행연구 비교의 지위

'통제(controlled)'라는 표현은 우리 데이터·프로토콜에서 우리가 직접 재채점한 경우에만 쓴다.

| 구분 | 대상 | 표기 |
|---|---|---|
| 통제 비교 | ESA CCI (우리가 우리 실측·프로토콜로 직접 재채점) | '통제'로 표기 가능 |
| 자체 비교 | co-kriging · 물리(Stefan/Kudryavtsev) 등 우리 데이터 내 자체 실행 | '우리 데이터 내 자체 비교'로 표기 |
| 보고값 병기 | Whitcomb 등 선행연구 문헌 수치 | '통제' 아님. 보고값 + 원 프로토콜(지역·CV·라벨 정의) 병기 |

원칙:
- 선행연구 문헌 수치는 그 연구의 평가 프로토콜(대상 지역, CV 방식, 라벨 정의, 기간)을 함께 적는다.
  프로토콜이 다르면 수치 직접 비교는 하지 않고 맥락 주석으로만 둔다.
- "SOTA 대비 우위" 같은 표현은 통제 재채점(ESA CCI)에 한정한다. 그 외에는 "보고값 대비 참고"로 한정한다.

## 4. 체크리스트 (보고 전 확인)

- [ ] 헤드라인 수치에 지표 라벨(공간블록/LORO)이 붙어 있는가.
- [ ] LORO 수치에 held-out 지역이 명시되어 있는가.
- [ ] 16.95cm을 '전이'로 잘못 부르고 있지 않은가(공간블록 CV로 표기).
- [ ] 절대 RMSE 옆에 R²·skill이 병기되어 있는가.
- [ ] '통제 비교'가 ESA CCI 재채점에만 쓰였는가.
- [ ] 선행연구 문헌 수치에 원 프로토콜이 병기되어 있는가.
